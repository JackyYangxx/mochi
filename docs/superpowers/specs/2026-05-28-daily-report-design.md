# 每日日报生成系统设计

## 概述

支持用户每天生成日报，AI 总结当日完成事项并对未完成事项给出建议。日报和归档分别保存为 MD 文件。

## 功能需求

### 触发方式

- **自动触发**：reminderTimes 到点时生成日报
- **手动触发**：Settings 面板添加"生成日报"按钮

### 数据获取

获取当天 0:00-23:59 的所有待办：
- 已完成事项
- 未完成事项

### 日报内容结构 (`YYYY-MM-DD-日报.md`)

```markdown
# YYYY-MM-DD 日报

## 完成事项
- item 1 ✓
- item 2 ✓

## 未完成事项
- item 3 → 建议...
- item 4 → 建议...

## 总结
（AI 生成的中文总结）
```

### 归档内容结构 (`archive.md`)

```markdown
# 待办归档

## 2026-05-28
- item 1 (完成) - HH:mm
- item 2 (完成) - HH:mm

## 2026-05-27
- ...
```

### 归档时机

日报生成成功后，自动归档当天已完成事项到 `archive.md`，并从数据库删除。

### 通知

生成完成后，在宠物图标附近显示 tips 提示。

## 配置项

| 配置项 | 说明 |
|--------|------|
| `reportDir` | 日报保存目录（Settings 配置） |

## 架构设计

| 模块 | 职责 |
|------|------|
| `DailyReportService` | 定时/手动触发、协调生成流程 |
| `LLMService` | 调用 LLM 生成日报内容 |
| `TodoService` | 获取当天待办、归档已完成事项 |
| `SettingsService` | 读写报告目录配置 |

## 数据流

```
触发 → 获取当天todos → 构造prompt → LLM生成 → 保存日报MD → 归档已完成 → tips通知 → 界面移除已完成项
```

## LLM Prompt 结构

```
请分析以下待办事项，生成日报：

已完成事项：
- item 1
- item 2

未完成事项：
- item 3
- item 4

请生成：
1. 每个未完成事项后的建议
2. 总结（中文）
```

## 实现步骤

1. Settings 添加日报目录配置和手动生成按钮
2. TodoService 添加按日期获取待办、归档方法
3. DailyReportService 新建，协调流程
4. 宠物旁 tips 通知组件
5. 集成到 ReminderService 定时触发
